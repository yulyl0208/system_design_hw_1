### Описание системы
Система позволяет пользователям вести учет тренировок, упражнений и получать статистику. Основные сущности: Пользователь, Тренировка, Упражнение.

### Роли пользователей
- Пользователь – физическое лицо, использующее приложение для создания тренировок, добавления упражнений, просмотра истории и статистики.

### Внешние системы
- Email Service – отправка email-уведомлений
- Push Notification Service – отправка push-уведомлений 

### Архитектура системы
Система построена по микросервисной архитектуре с API Gateway, отдельными сервисами для каждой предметной области, асинхронным взаимодействием через брокер сообщений и кэшированием.

### Взаимодействие контейнеров
- Клиент общается с API Gateway по HTTPS/REST.
- API Gateway перенаправляет запросы к соответствующим сервисам по gRPC.
- Сервисы читают/пишут в БД через JDBC, используют кэш Redis.
- Сервисы публикуют события в Kafka 
- Notification Consumer подписан на события и отправляет уведомления через внешние сервисы по HTTPS.

### Основные сценарии использования 
- Создание нового пользователя
- Создание упражнения
- Создание тренировки
- Получение статистики тренировок за период

Последовательность шагов:
1. Пользователь запрашивает статистику за период.
2. Mobile App отправляет GET /statistics?period=... на API Gateway.
3. API Gateway вызывает Statistics Service по gRPC.
4. Statistics Service проверяет наличие данных в Redis.
5. При отсутствии (cache miss) выполняет агрегирующий запрос к PostgreSQL.
6. Сохраняет результат в Redis.
7. Возвращает данные через API Gateway в приложение.
8. Statistics Service публикует событие в Kafka.
9. Notification Consumer получает событие и отправляет push-уведомление.
